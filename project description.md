# AI Chef — Описание проекта

## 1. Краткое бизнес-описание

**AI Chef** — мобильное приложение для генерации персонализированных рецептов на основе доступных у пользователя продуктов.

**Основная ценность:** Пользователь вводит список продуктов (текстом, голосом или фото), система подбирает подходящие рецепты из базы данных или генерирует уникальный рецепт через AI.

**Целевая аудитория:** Старшее поколение (требует интуитивный интерфейс и обучающие материалы).

**Бизнес-модель:** Freemium с подписками и покупкой дополнительных генераций.

**География:** Узбекистан (платежные системы Click, Payme).

**Технологии:**

- Backend: Node.js + Fastify + PostgreSQL
- AI: OpenAI (GPT-4.1, Whisper)
- Языки: русский, узбекский, английский

---

## 2. Основной функционал

### Ввод данных

- **Текст** — список продуктов или название блюда
- **Голос** — транскрипция через Whisper API
- **Фото** — распознавание продуктов через GPT-4.1 Vision
- **Комбо** — фото + текстовое уточнение

### Поиск рецептов

- **Быстрый поиск** — по кураторской базе данных (детерминированный, мгновенный)
- **Уникальная генерация** — персонализированный рецепт через LLM (медленнее, дороже)

### Дополнительные функции

- **История** — сохранение всех поисков и результатов
- **Избранное** — сохранение понравившихся рецептов
- **Подписки** — управление генерациями и оплатой
- **Админка** — управление каталогом, настройками, платежами

## 3. Как работают основные функции

### 3.1 Поиск по каталогу (основной флоу)

```
Пользователь вводит данные
        ↓
Нормализация через LLM
(извлечение ингредиентов и блюд)
        ↓
Поиск в БД по двум критериям:
1. Точное совпадение названия блюда
2. Ранжирование по ингредиентам
        ↓
Выдача 3 рецептов
(списание 1 генерации)
        ↓
Локализация под язык UI
        ↓
Результат пользователю
```

**Пример:**

- Ввод: "Хочу плов, есть рис и морковка"
- Нормализация: `ingredients: ["rice", "carrot", "lamb", "onion"]`, `dishes: ["pilaf"]`
- Поиск: находит "Плов узбекский", "Паэлья", "Ризотто"
- Ранжирование: плов выше (совпадение по названию + больше общих ингредиентов)

**Алгоритм ранжирования:**

```
score = 2*(совпадения/размер_рецепта)
      + 1*(совпадения/продукты_пользователя)
      - 1.5*(недостающие/размер_рецепта)
      + бонус_за_название
```

**Важно:**

- Нормализация для каталога всегда на английском (независимо от языка UI)
- Ответ локализуется под язык пользователя
- Это позволяет вводить запрос на любом языке

### 3.2 Уникальная генерация

```
Пользователь нажимает
"Сгенерировать уникальный рецепт"
        ↓
Нормализация через LLM
(на языке пользователя)
        ↓
Полная генерация рецепта через GPT
(название, ингредиенты, шаги, БЖУ, фото)
        ↓
Сохранение в отдельную таблицу
(только для этого пользователя)
        ↓
Списание 1 генерации
        ↓
Результат с визуальным выделением
```

**Особенности:**

- Генерация занимает ~30 секунд
- Показывается прогресс-бар и полезные советы
- Уникальные рецепты видны только их создателю
- Могут добавляться в избранное и историю

### 3.3 История генераций

**Структура:**

- **Сессия** — один поисковый запрос пользователя
- **Записи** — все рецепты, полученные в рамках этой сессии

**Что входит в одну сессию:**

- Первые 3 рецепта из каталога
- Все дополнительные рецепты из каталога (пагинация)
- Все уникальные рецепты для этого запроса

**Новая сессия создается когда:**

- Изменился текст запроса
- Загружено новое фото
- Новый голосовой ввод

### 3.4 Система подписок

**Валюта:** `remaining_generations` — количество доступных генераций

**Что списывает генерации:**

- Первый поиск по каталогу (3 рецепта) — 1 генерация
- Пагинация (еще 3 рецепта) — 1 генерация
- Уникальный рецепт — 1 генерация

**Тарифы:**

- **Free** — периодическое пополнение бесплатных генераций
- **Premium** — месячная подписка с пакетом генераций
- **Докупка** — разовая покупка генераций

**Платежные системы:** Click и Payme (Узбекистан)

### 3.5 Многоязычность

**Принцип (важно!):**

- Индекс поиска — англоязычный
- Нормализация для каталога — всегда на английском
- Ответ API — локализуется под `request.lang`

**Зачем:**
Пользователь может выбрать узбекский интерфейс, но ввести запрос по-русски — поиск все равно сработает.

**Поддерживаемые языки:** русский, узбекский, английский

## 4. Особенности проекта

### 4.1 Гибридная модель

- 90%+ запросов обслуживаются из БД (быстро, дешево)
- 10%- через LLM (медленно, дорого, но персонализировано)
- Оптимальный баланс скорости, качества и стоимости

### 4.2 Языковая независимость

- Англоязычный поисковый индекс
- Ввод на любом языке → поиск работает
- Результат локализуется автоматически

### 4.3 Интеллектуальное ранжирование

- Учет множества факторов: совпадения, недостающие, название
- Приоритизация рецептов с максимальным использованием имеющихся продуктов

### 4.4 Атомарность операций

- Списание генераций в транзакциях
- Идемпотентность платежных callbacks
- Неразрушающие миграции каталога

### 4.5 Фоновая обработка

- Генерация продолжается при скрытии приложения
- Автоматическое продление подписок (cron)
- Начисление бесплатных генераций (cron)

## 6. Основные API endpoints

### Аутентификация

- `POST /auth/otp` — отправка OTP
- `POST /auth/validate-otp` — проверка OTP → JWT

### Рецепты

- `POST /task/search-catalog` — поиск по каталогу
- `POST /task/generate-recipe` — уникальная генерация

### Пользователь

- `GET /history` — история генераций
- `GET /subscriptions/status` — баланс и статус подписки
- `PATCH /profile/onboarding` — статус обучения

### Платежи

- `POST /payment/initiate` — инициация платежа
- `POST /click`, `POST /payment/callback/payme` — callbacks

### Админка

- `GET /admin/recipes/panel` — управление каталогом
- `GET /admin/app-config` — глобальные настройки

### Служебные

- `GET /health` — проверка здоровья
- `GET /metrics` — Prometheus метрики
- `GET /docs` — Swagger UI

## 7. Структура БД (упрощенно)

### Пользователи

- `users` — телефон, профиль, `remaining_generations`, подписка
- `otp_codes` — OTP для входа

### Каталог

- `recipes_catalog` — рецепты с переводами (JSONB)
- `ingredients_dictionary` — словарь ингредиентов (canonical names)
- `recipes_catalog_ingredients` — индекс для поиска

### История

- `generation_sessions` — запросы пользователя
- `generation_entries` — рецепты в каждой сессии

### Коммерция

- `subscriptions`, `plans`, `user_generations`
- `click_card_tokens`, `click_payments`
- (аналогично для Payme)

### Управление

- `app_versions` — версии приложения
- `app_config` — глобальные флаги (maintenance, payments enabled)

## 10. Что важно знать

### Критичные моменты

1. **Язык поиска vs язык UI** — нормализация для каталога всегда на английском
2. **Атомарность** — списание генераций до выполнения операции
3. **Идемпотентность** — платежные callbacks должны обрабатываться повторно без ошибок
4. **Локализация** — DTO преобразуется в нужный язык перед отправкой

### Что нельзя ломать

- Тексты и переводы рецептов (только dictionary/mappings)
- DTO мобильных клиентов (обратная совместимость)
- Независимость поиска от языка UI

### Полезные команды

- `npm run dev` — запуск dev сервера
- `npm run build` — сборка для продакшена
- `npm run db:push` — применить изменения схемы
- `npm run db:studio` — UI для просмотра БД
